# Wiam Test App

Тестовое задание на позицию frontend-developer

## Запуск проекта

Вы можете запустить проект двумя способами: через Docker или локально с помощью pnpm.

### Вариант 1: Через Docker

Этот способ не требует установки Node.js и зависимостей на ваш компьютер. Достаточно иметь установленный Docker.

1.  **Сборка образа:**
    ```bash
    docker build -t wiam-test-app .
    ```

2.  **Запуск контейнера:**
    ```bash
    docker run -d --rm -p 8080:80 wiam-test-app
    ```

После этого приложение будет доступно по адресу: [http://localhost:8080](http://localhost:8080)

---

### Вариант 2: Локальный запуск (через pnpm)

Для этого способа вам понадобится установленный Node.js.

1.  **Установка pnpm** (если он еще не установлен):
    ```bash
    npm install -g pnpm
    ```

2.  **Установка зависимостей:**
    ```bash
    pnpm install
    ```

3.  **Сборка и запуск:**
    ```bash
    pnpm build && pnpm preview
    ```

После выполнения этих команд в консоли появится ссылка на локальный сервер (обычно [http://localhost:4173](http://localhost:4173)).

##  Библиотеки
*   react, react-router-dom - использование указано в ТЗ
*   react-bootstrap - допускается по ТЗ для стилизации и использования готовых компонентов
*   react-hook-form - стандарт при разработке форм на react, гибкое api, большое комьюнити
*   react-number-format - легковесная библиотека для работы с масками, если честно поленился писать свою маску:)

##  Решения
*   код разбит на атомарные компоненты, по принипу единой ответсвенности. Такой подход упростит написание юнит/интеграционных тестов 
*   почти весь текст вынесен в конфиг TEXTS. Для переиспользования в компонентах и тестов к ним. Конфиг - один источник истины 
*   немного оверинжиниринга с компонентом MultipageFormProvider. Не захотелось захламлять App инициализацией формы. Плюс, если в приложение добавлять другие роуты не будет ререндера всего приложения при изменениях в форме
